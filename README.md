# Student Support Admin Dashboard

An internal administrative dashboard for managing service requests, case progression, staff allocation, and record visibility in a student support environment.

## Project Purpose

This portfolio project simulates a realistic staff-facing case management system used by university or government-style administrative teams to process incoming student support requests. It is designed to showcase practical internal-tool UI patterns rather than public-facing product design.

## Target Users

- Student support officers
- Case coordinators
- Student administration and records staff
- Team leaders managing workload allocation
- Government or university digital teams working on intranet and operational systems

## Key Features

- Dashboard overview with workload metrics, recent activity, category distribution, SLA warnings, and priority cases
- Searchable, filterable request register with sorting, pagination, and row selection
- Detailed request side panel with student information, attached documents, internal notes, and activity log
- Local workflow actions for assignment, status updates, priority updates, resolution, reopening, and case note entry
- Assigned to Me, Teams / Staff, Reports, and Settings sections to demonstrate broader internal system structure
- TypeScript-based component architecture with reusable badges, metric cards, filters, table, timeline, and activity components

## Why It Is Relevant To Administrative And Government-Style Roles

The project focuses on the kind of structured, traceable, accessibility-conscious interface patterns commonly used in internal education and public-sector systems:

- case tracking and triage
- clear record keeping and audit visibility
- workload management across staff
- operational reporting
- semantic structure and keyboard-visible focus states
- table-heavy interfaces designed for clarity and speed
